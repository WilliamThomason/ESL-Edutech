# Translation Workflow Design — Eloquenta ## 1. L1 Translation Delivery Process ### Philosophy The L1 (native language) translation is a **scaffold**, not a crutch. Students should hear and speak English first, then see the L1 translation as confirmation — never as the starting point. ### Translation Delivery Flow ``` ┌─────────────────────────────────────────────────────────────┐ │ LESSON START │ │ Teacher speaks English sentence aloud │ │ Students listen → repeat → practice │ │ (NO translation visible yet) │ └─────────────────────┬───────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────┐ │ CHECK UNDERSTANDING │ │ Teacher asks concept-check questions in English │ │ Students respond in English (or L1 if stuck) │ │ If >40% students struggle → show L1 translation │ └─────────────────────┬───────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────┐ │ L1 TRANSLATION REVEAL │ │ Translation appears with phonetic guide │ │ Students compare their understanding to the translation │ │ Teacher highlights false friends / tricky constructions │ └─────────────────────┬───────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────┐ │ PRACTICE WITH TRANSLATION HIDDEN │ │ Translation fades / is hidden │ │ Students produce English from memory │ │ (Spaced repetition — translation reappears at intervals) │ └─────────────────────────────────────────────────────────────┘ ``` ### Translation Trigger Rules | Situation | Action | |---|---| | New vocabulary (>3 unknown words) | Show L1 before speaking | | Idiom / figurative language | Show L1 after first failed attempt | | Grammar structure not yet taught | Show L1 alongside English | | Student requests translation | Show L1 individually (not whole class) | | >40% comprehension failure | Show L1 to whole class | | Review / warm-up | Hide L1, test from memory | ### Translation Display Modes 1. **Inline hover** — Student hovers over a word/phrase to see L1 2. **Tap to reveal** — Student taps sentence to toggle L1 3. **Teacher-controlled** — Translation appears only when teacher clicks 4. **Timed fade** — Translation appears for 5 seconds, then fades 5. **Side-by-side** — English left, L1 right (for complex explanations) --- ## 2. AI Tool Comparison & Recommendation ### DeepL API - **Strengths:** Best quality for European languages, handles nuance and idioms well, formal/informal tone control, glossary support for technical terms - **API:** REST API, 500K chars/month free, €8.99/month for more - **Languages:** 30+ European and Asian languages - **Best for:** French, German, Spanish, Italian, Portuguese, Dutch, Polish, Russian, Japanese, Chinese - **Cost:** ~$0.02 per 1000 characters via API translator skill - **Integration:** `deepl` npm package or direct REST calls ### Reverso Context - **Strengths:** Real-world example sentences, shows translations in context, good for idiomatic expressions - **API:** Limited free tier, paid plans for high volume - **Languages:** 14 languages - **Best for:** Students who need to see words in real context, not just direct translations - **Integration:** Context reverso API (less robust than DeepL) ### Google Translate API - **Strengths:** Broadest language coverage (133+ languages), cheapest for high volume, batch translation - **API:** $20 per 1M characters, generous free tier - **Languages:** 133+ - **Best for:** Less common languages (Korean, Thai, Vietnamese, Hindi, Arabic, etc.) - **Integration:** `@google-cloud/translate` npm package ### Recommendation: Hybrid Approach ``` Primary: DeepL (European languages, best quality) Fallback: Google Translate (non-European languages, broadest coverage) Enrich: Reverso Context (example sentences for vocabulary lessons) ``` ### Implementation ```javascript // Translation service with fallback chain async function translate(text, targetLang, sourceLang = 'EN') { try { // Try DeepL first (best quality for supported languages) return await translateWithDeepL(text, targetLang, sourceLang); Eloquence-First Lesson Structure ### Core Principle **Speak first, grammar second.** Students learn to communicate before they learn to be correct. Grammar emerges from communication, not the other way around. ### Lesson Architecture (45-minute lesson) ``` ┌─────────────────────────────────────────────────────────────┐ │ PHASE 1: WARM-UP (5 min) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ • Greet in English (no L1) │ │ │ │ • Quick review: "What did we learn last time?" │ │ │ │ • Teacher models 2-3 key phrases from last lesson │ │ │ │ • Students repeat (chorus + individual) │ │ │ │ • NO grammar explanation yet │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ PHASE 2: NEW CONTENT — SPEAK FIRST (15 min) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ • Teacher introduces topic with visual context │ │ │ │ • Teacher speaks target phrases naturally │ │ │ │ • Students listen → repeat → practice in pairs │ │ │ │ • L1 translation ONLY if >40% fail comprehension │ │ │ │ • Focus: Can you SAY it? (not "can you explain it?") │ │ │ │ • Error correction: pronunciation > grammar │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ PHASE 3: GUIDED PRACTICE (10 min) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ • Role-play / scenario practice │ │ │ │ • Students use target language in context │ │ │ │ • Teacher circulates, corrects pronunciation │ │ │ │ • L1 available on demand (student taps to reveal) │ │ │ │ • Peer correction encouraged │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ PHASE 4: GRAMMAR SPOTLIGHT (5 min) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ • NOW explain the grammar pattern │ │ │ │ • Use examples students already spoke │ │ │ │ • "You just said X. Here's why it works that way." │ │ │ │ • L1 translation for grammar explanation if needed │ │ │ │ • Keep it under 5 minutes — don't over-explain │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ PHASE 5: FREE PRACTICE + CLOSING (10 min) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ • Open conversation using target language │ │ │ │ • No L1 visible (translation hidden) │ │ │ │ • Teacher notes errors for next lesson │ │ │ │ • Closing: Students summarize what they learned │ │ │ │ • Preview next lesson's topic │ │ │ └─────────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────────┘ ``` ### Translation Integration Points | Phase | Translation Visible? | How | |---|---|---| | Warm-up | ❌ No | Pure English review | | New content | ⚠️ Conditional | Only if comprehension <60% | | Guided practice | ✅ On demand | Student taps to reveal | | Grammar spotlight | ✅ Yes | L1 for grammar explanation | | Free practice | ❌ No | English only | ### Lesson Material Template Each lesson page should have this structure: ```html
